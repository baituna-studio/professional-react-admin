## Why
Users management is a key admin workflow. The template needs a usable, interactive users page to demonstrate table-driven operations and detail inspection.

## What Changes
- Build users page with search, status filter, pagination, and row actions.
- Add user detail drawer/modal interaction from row selection.
- Add empty and loading states integrated with query and filters.

## Capabilities

### New Capabilities
- `users-management-ui`: Interactive users listing and inspection workflow.

### Modified Capabilities
- None.

## Impact
- Affects `features/users` module, shared table/status components, and route page behavior.
- Demonstrates practical admin CRUD-adjacent UX without backend complexity.
